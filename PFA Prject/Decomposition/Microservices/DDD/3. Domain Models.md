

### 1. Identity & Access Management Context
Provides user identity and authentication to all other contexts

```mermaid
classDiagram
    class UserId {
        <<Value Object>>
        String value
    }
    
    class User {
        <<Entity>>
        UserId id
        String email
        String username
        String hashedPassword
        UserStatus status
        List~Role~ roles
    }
    
    class UserStatus {
        <<Value Object>>
        ACTIVE
        INACTIVE
        DISABLED
    }
    
    class Role {
        <<Value Object>>
        ANONYMOUS
        AUTHENTICATED
        ADMIN
    }
    
    class UserAggregate {
        <<Aggregate>>
        User root
        UserProfile profile
    }
    
    class UserProfile {
        <<Entity>>
        UserId userId
        String displayName
        String bio
        String avatarUrl
        DateTime createdAt
        DateTime lastLoginAt
    }
    
    class AuthenticationToken {
        <<Value Object>>
        String token
        UserId userId
        DateTime issuedAt
        DateTime expiresAt
    }
    
    class UserService {
        <<Domain Service>>
        registerUser(email, username, password)
        deactivateUser(userId)
        changeUserRole(userId, role)
    }
    
    class AuthenticationService {
        <<Domain Service>>
        authenticate(email/username, password)
        validateToken(token)
        revokeToken(token)
    }
    
    class UserApplicationService {
        <<Application Service>>
        register(RegisterUserCommand)
        login(LoginUserCommand)
        updateProfile(UpdateProfileCommand)
        deactivate(DeactivateUserCommand)
        logout(LogoutUserCommand)
    }
    
    UserAggregate --* User : contains
    UserAggregate --* UserProfile : contains
    User --o UserId : has
    User --o UserStatus : has
    User --o Role : has many
    UserProfile --o UserId : references
    AuthenticationToken --o UserId : references
    UserService --> UserAggregate : operates on
    AuthenticationService --> UserAggregate : operates on
    AuthenticationService --> AuthenticationToken : manages
    UserApplicationService --> UserService : uses
    UserApplicationService --> AuthenticationService : uses
```

Organizes content and manages subscriptions

```mermaid

    }
    
    class UserId {
        <<Value Object>>
        String value
    }

        UserId ownerId

    }
    
    class ChannelStatus {
        <<Value Object>>

        INACTIVE

    }

        <<Value Object>>
```

Handles videos, articles, comments and chat messages

```mermaid

        <<Value Object>>
        String value

        <<Value Object>>
        String value
    }

        String value
    }

        ChannelId channelId
        UserId creatorId
        String title
        String description
        VideoType type
        String thumbnailUrl
        String videoUrl
        Integer duration
        Integer viewCount
        DateTime createdAt
        DateTime publishedAt
    }
    
    class VideoType {
        <<Value Object>>
        LIVE
        VOD
    }
    
    class Article {
        <<Entity>>

        UserId authorId
        String title
        String content
        ArticleCategory category
        List~String~ tags

        DateTime createdAt
        DateTime publishedAt

    class ArticleCategory {
        <<Value Object>>

        NEWS
        GUIDE
    }
    
    class Comment {
        <<Entity>>
        String id

        String text

    ChatMessage --o UserId : sent by

    ContentService --> ArticleAggregate : operates on
    ContentQueryService --> VideoAggregate : reads
    ContentQueryService --> ArticleAggregate : reads

    ContentApplicationService --> ContentQueryService : uses
```

### 4. Notification Context
Responds to events from other contexts to create notifications
```mermaid
classDiagram
    class NotificationId {
        <<Value Object>>
        String value
    }
    
    class UserId {
        <<Value Object>>
        String value
    }
    
    class Notification {
        <<Entity>>
        NotificationId id
        UserId recipientId

    class NotificationType {
        <<Value Object>>

        <<Aggregate>>

    class NotificationMetadata {
        <<Value Object>>

    }
    
    class NotificationPreference {
        <<Entity>>

    }
    
    class NotificationService {
        <<Domain Service>>
        createNotification(recipientId, type, data)
```

### 5. Analytics Context
Tracks metrics across all contexts for reporting
```mermaid
classDiagram
    class MetricId {
        <<Value Object>>
        String value
    }
    
    class EntityId {
        <<Value Object>>
        String value
        String entityType
    }
    
    class Metric {
        <<Entity>>
        MetricId id
        String name
        MetricType type
        EntityId entityId
        Double value
        DateTime timestamp
    }
    
    class MetricType {
        <<Value Object>>
        VIEW_COUNT
        SUBSCRIBER_COUNT
        COMMENT_COUNT
        ACTIVE_USERS
        LIVE_VIEWERS
        NEW_REGISTRATIONS
    }
    
    class MetricAggregate {
        <<Aggregate>>
        List~Metric~ metrics
        TimeRange timeRange
    }
    
    class TimeRange {
        <<Value Object>>
        DateTime from
        DateTime to
        TimeGranularity granularity
    }
    
    class TimeGranularity {
        <<Value Object>>
        MINUTE
        HOUR
        DAY
        WEEK
        MONTH
    }
    
    class DashboardStatistics {
        <<Value Object>>
        Integer totalUsers
        Integer totalChannels
        Integer totalVideos
        Integer totalArticles
        Integer activeStreams
        List~DailyStatistic~ dailyStats
    }
    
    class DailyStatistic {
        <<Value Object>>
        Date day
        Integer newUsers
        Integer activeUsers
        Integer contentViews
    }
    
    class ChannelStatistics {
        <<Value Object>>
        EntityId channelId
        Integer subscriberCount
        Integer videoCount
        Integer articleCount
        Integer totalViews
        List~MetricAggregate~ metricTrends
    }
    
    class ContentStatistics {
        <<Value Object>>
        EntityId contentId
        Integer viewCount
        Integer commentCount
        Integer engagementScore
        Map~String, Double~ demographics
    }
    
    class AnalyticsService {
        <<Domain Service>>
        trackMetric(entityId, metricType, value)
        aggregateMetrics(entityIds, metricTypes, timeRange)
    }
    
    class AnalyticsQueryService {
        <<Domain Service>>
        getDashboardStatistics(timeRange)
        getChannelStatistics(channelId, timeRange)
        getContentStatistics(contentId)
        getGrowthTrends(metricType, timeRange)
    }
    
    class AnalyticsApplicationService {
        <<Application Service>>
        trackEvent(TrackEventCommand)
        getDashboardStatistics(DashboardQuery)
        getEntityStatistics(EntityStatsQuery)
        generateReport(ReportQuery)
    }
    
    class AnalyticsEventHandler {
        <<Domain Service>>
        handleUserRegistered(event)
        handleContentViewed(event)
        handleChannelSubscribed(event)
        handleContentCreated(event)
        handleStreamStarted(event)
        handleStreamEnded(event)
    }
    
    MetricAggregate --* Metric : contains many
    MetricAggregate --* TimeRange : spans
    Metric --o MetricId : has
    Metric --o MetricType : has
    Metric --o EntityId : measures
    TimeRange --o TimeGranularity : has
    
    DashboardStatistics --* DailyStatistic : contains many
    ChannelStatistics --o EntityId : references
    ChannelStatistics --* MetricAggregate : contains many
    ContentStatistics --o EntityId : references
    
    AnalyticsService --> MetricAggregate : operates on
    AnalyticsQueryService --> MetricAggregate : reads
    AnalyticsQueryService --> DashboardStatistics : creates
    AnalyticsQueryService --> ChannelStatistics : creates
    AnalyticsQueryService --> ContentStatistics : creates
    AnalyticsApplicationService --> AnalyticsService : uses
    AnalyticsApplicationService --> AnalyticsQueryService : uses
    AnalyticsEventHandler --> AnalyticsService : uses
```
